# アイデア評価基準（5軸）

## 概要

ヒアリング結果から LLM が自動評価し、requirements.md 内に記載する。
ユーザーへの追加質問は不要。ヒアリング回答から導出する。

**注意**: スコアは情報提示であり、命令的な判定（「3.0未満は中止」等）は行わない。

## 各軸の評価基準

### 1. タイミング（市場機会の時間軸）

| スコア | 基準 |
|--------|------|
| 5 | 規制変更・プラットフォーム変更など、今しかない明確なウィンドウがある |
| 4 | 市場トレンドが追い風。半年以内に参入すべき理由がある |
| 3 | 特に急ぐ理由はないが、需要は存在する |
| 2 | 市場が成熟しており、後発参入のメリットが薄い |
| 1 | 市場が縮小傾向、またはタイミングが早すぎる |

### 2. 実現可能性（技術的難度・MVP期間）

| スコア | 基準 |
|--------|------|
| 5 | 既存ライブラリ・APIの組み合わせで1週間以内にMVP完成 |
| 4 | 標準的な技術スタックで2週間以内にMVP完成 |
| 3 | 一部カスタム実装が必要だが、1ヶ月以内にMVP完成 |
| 2 | 複雑なインフラ・専門技術が必要。MVP完成に1ヶ月以上 |
| 1 | 研究開発レベルの技術が必要、または外部依存が多すぎる |

### 3. 収益性（月額モデルでの収入予測）

| スコア | 基準 |
|--------|------|
| 5 | 明確な課金モデル。100ユーザーで月$1,000以上が見込める |
| 4 | 課金モデルが成立する。100ユーザーで月$500程度 |
| 3 | フリーミアム等で収益化可能だが、無料ユーザーが多くなりそう |
| 2 | 収益化の道筋が不明瞭。広告モデルに依存 |
| 1 | 収益化が極めて困難。無料でしか提供できない |

### 4. 競合優位性（ニッチの明確性）

| スコア | 基準 |
|--------|------|
| 5 | 競合がほぼ存在しないニッチ。明確な差別化がある |
| 4 | 競合はいるが、特定セグメントで明確な優位性がある |
| 3 | 競合が数社いるが、差別化ポイントは説明できる |
| 2 | 競合が多く、差別化が「UIが良い」程度に留まる |
| 1 | レッドオーシャン。大手が支配する市場 |

### 5. 個人開発適性（サポートコスト・運用負荷）

| スコア | 基準 |
|--------|------|
| 5 | セルフサービス完結。サポート不要。サーバーレスで運用コスト最小 |
| 4 | 軽微なサポートのみ。月数時間の運用で回る |
| 3 | 定期的なコンテンツ更新やモデレーションが必要 |
| 2 | ユーザー間トラブル対応やリアルタイム監視が必要 |
| 1 | 24/7対応が必要、または法的リスクが高い領域 |

## 総合スコアの読み方

- **4.0以上**: 個人開発に非常に適している。すぐに着手推奨
- **3.0〜3.9**: 概ね良好。低スコアの軸を補強すればさらに良くなる
- **2.0〜2.9**: リスクがある軸が複数。ピボットや絞り込みを検討
- **2.0未満**: 根本的な見直しを推奨

※ あくまで参考指標。最終判断はユーザーが行う。
