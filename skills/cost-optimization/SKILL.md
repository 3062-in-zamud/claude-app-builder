# cost-optimization: インフラコスト最適化計画

## 概要

プロダクトのインフラコストを棚卸しし、収益性（Gross Margin）を改善するための具体的な施策を策定する。各SaaSの代替比較、コスト削減チェックリスト、月次レビュープロセスを含む実践的なコスト最適化計画を生成する。

## ワークフロー

### Step 1: プロジェクトコンテキスト読み込み

以下のファイルが存在すれば読み込み、現在の技術スタックとコスト構造を把握する:

- `docs/tech-stack.md`
- `docs/pricing-strategy.md`
- `docs/scaling-strategy.md`

### Step 3: コスト/MRR比率目標設定

収益に対するインフラコストの健全性を評価する:

#### コスト比率の目安
| 指標 | 健全 | 注意 | 危険 |
|------|------|------|------|
| インフラコスト / MRR | < 30% | 30-50% | > 50% |
| Gross Margin | > 70% | 50-70% | < 50% |

#### 計算式
```
Gross Margin = (MRR - インフラコスト) / MRR × 100

例:
MRR = $500, インフラコスト = $100
Gross Margin = ($500 - $100) / $500 × 100 = 80% ✅ 健全
```

#### MRR別のインフラコスト上限目安
| MRR | インフラ上限（30%） | 推奨構成 |
|-----|-------------------|---------|
| $0 (プレローンチ) | $0-20 | 全サービスFree tier |
| $100 | $30 | Supabase Pro + Vercel Hobby |
| $500 | $150 | Supabase Pro + Vercel Pro |
| $1,000 | $300 | + Sentry Team + Redis |
| $5,000 | $1,500 | + 専用DB + CDN最適化 |
| $10,000+ | $3,000 | マルチリージョン検討 |

### Step 5: Gross Margin改善計画

Gross Margin ≥ 70% を達成・維持するための具体的施策:

#### 即効性のある施策（1週間以内）
1. **未使用リソースの停止**: 使っていない環境・サービスの削除
2. **プランダウングレード**: 利用量に対してオーバースペックなプランの見直し
3. **Free Tierの最大活用**: 各サービスのFree枠を使い切っているか確認
4. **ログ保持期間の短縮**: 不要に長いログ保持を短縮

#### 中期施策（1-3ヶ月）
1. **キャッシュ最適化**: CDN/ブラウザキャッシュ改善でサーバーコスト削減
2. **画像最適化**: Next.js Image + WebP/AVIF変換で帯域削減
3. **コネクションプーリング**: DB接続効率化でプラン維持
4. **バンドルサイズ削減**: 不要な依存関係の削除

#### 長期施策（3-6ヶ月）
1. **サービス移行**: コスト効率の良い代替サービスへの段階的移行
2. **アーキテクチャ最適化**: Edge Computing活用、SSG/ISR拡大
3. **年間契約割引**: 安定利用のサービスは年契約で10-20%削減
4. **セルフホスト検討**: 規模が大きくなったサービスのセルフホスト化

### Step 6: コスト削減チェックリスト

月次レビューで使用するチェックリスト:

#### インフラ全般
- [ ] 未使用のVercelプロジェクト・デプロイがないか
- [ ] Supabaseの未使用テーブル・ストレージがないか
- [ ] 開発・ステージング環境が不要に稼働していないか
- [ ] 期限切れのドメイン・SSL証明書がないか

#### Vercel
- [ ] Serverless Function の実行時間は適切か（タイムアウト設定）
- [ ] 帯域使用量はプランの上限内か
- [ ] Edge Functions で代替可能な Serverless Function はないか
- [ ] 不要なPreview Deploymentが蓄積していないか

#### Supabase
- [ ] データベースサイズはプランの上限内か
- [ ] Storage の未使用ファイルを削除したか
- [ ] Realtime の不要なサブスクリプションがないか
- [ ] Auth のMAU（月間アクティブユーザー）はプランの上限内か
- [ ] Edge Functions の実行回数は適切か
- [ ] コネクションプーリングが有効化されているか

#### Stripe
- [ ] 手数料率は最適か（規模に応じた交渉余地の確認）
- [ ] 不要なWebhookエンドポイントがないか
- [ ] テスト環境のデータが蓄積していないか

#### 外部サービス
- [ ] 各SaaSのFree Tier上限に達しそうか（事前にプラン移行検討）
- [ ] 使っていないSaaSサブスクリプションがないか
- [ ] 年間契約に切り替えたほうが安いサービスはないか
- [ ] セルフホスト版に移行したほうが安いサービスはないか

#### コード・アーキテクチャ
- [ ] 不要な外部APIコールを削減できないか
- [ ] キャッシュヒット率は十分か（> 80%目標）
- [ ] バンドルサイズが増加傾向にないか
- [ ] N+1クエリが発生していないか
