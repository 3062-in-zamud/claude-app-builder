# CRO LIFT Model チェックリスト

## LIFT Model とは

WiderFunnel社が開発したコンバージョン最適化フレームワーク。
6つの要素でランディングページの転換率を体系的に改善する。

```
コンバージョン = Value Proposition + Relevance + Clarity - Anxiety - Distraction
                                                          ↑ Urgency で後押し
```

## 1. Value Proposition（価値提案）

**目的**: 訪問者が得られる価値を明確に伝える

### チェックリスト
- [ ] ヘッドラインが「何を得られるか」を具体的に伝えている
- [ ] 競合との差別化ポイントが明示されている
- [ ] 数値で価値を表現している（例: 「作業時間を80%削減」）
- [ ] ベネフィットがフィーチャーより先に来ている

### 実装パターン
```tsx
// NG: フィーチャー先行
<h1>AIパワードのタスク管理ツール</h1>

// OK: ベネフィット先行
<h1>毎日2時間のムダを取り戻す</h1>
<p>AIが優先順位を自動判定。重要なことだけに集中できます</p>
```

## 2. Relevance（関連性）

**目的**: 訪問者が「自分のためのサービスだ」と感じる

### チェックリスト
- [ ] ターゲットの課題・痛みを具体的に言語化している
- [ ] 訪問者のコンテキスト（流入元）に合ったメッセージになっている
- [ ] 業界・職種に特化した表現を使っている
- [ ] ペルソナの言葉遣いに寄せたコピーになっている

### 実装パターン
```tsx
// ペルソナに応じた表現の使い分け
// エンジニア向け: 「CLI一発で環境構築完了」
// マネージャー向け: 「チームの生産性を可視化」
// デザイナー向け: 「デザインシステムと自動同期」
```

## 3. Clarity（明確性）

**目的**: メッセージに曖昧さがなく、一読で理解できる

### チェックリスト
- [ ] ヘッドラインが15文字以内で要点を伝えている
- [ ] 専門用語を避けている（または即座に説明している）
- [ ] 視覚的なヒエラルキーが明確（見出し > サブ > 本文）
- [ ] CTAボタンのテキストが「何が起きるか」を伝えている
- [ ] 1セクション1メッセージの原則を守っている

### 実装パターン
```tsx
// NG: 曖昧なCTA
<Button>詳しくはこちら</Button>

// OK: 具体的なCTA
<Button>無料で14日間試す</Button>
<p className="text-sm text-gray-500">クレジットカード不要</p>
```

## 4. Urgency（緊急性）

**目的**: 今すぐ行動する理由を作る

### チェックリスト
- [ ] 期間限定オファーがある場合、期限を明示している
- [ ] 残り枠数や限定数を表示している（嘘でない範囲で）
- [ ] 「後でやろう」のコストを伝えている
- [ ] 行動しない場合の機会損失を示唆している

### 実装パターン
```tsx
// 内発的緊急性（課題の深刻さを伝える）
<p>毎日15分のロスが、年間91時間の損失に</p>

// 外発的緊急性（オファーの期限）
<div className="bg-yellow-50 p-4 rounded">
  <p>アーリーアダプター特典: 先着100名まで永年20%OFF</p>
  <p className="text-sm">残り{count}枠</p>
</div>
```

## 5. Anxiety（不安）

**目的**: 行動を妨げる不安要素を除去する

### チェックリスト
- [ ] 「無料トライアルにクレジットカード不要」を明示
- [ ] プライバシーポリシーへのリンクがCTA近くにある
- [ ] セキュリティバッジ・SSL証明を表示
- [ ] 返金保証・キャンセルの容易さを伝えている
- [ ] FAQで主要な懸念に回答している

### 実装パターン
```tsx
<div className="flex items-center gap-2 mt-4 text-sm text-gray-500">
  <ShieldCheckIcon className="h-4 w-4" />
  <span>256bit SSL暗号化 | いつでもキャンセル可能 | クレカ不要</span>
</div>
```

## 6. Distraction（気を散らす要素）

**目的**: コンバージョンに無関係な要素を排除する

### チェックリスト
- [ ] ナビゲーションリンクが最小限（LPでは非表示推奨）
- [ ] 外部リンクがない（離脱ポイント排除）
- [ ] CTAが1種類に絞られている（選択のパラドックス回避）
- [ ] 不要な画像・アニメーションがない
- [ ] フォームフィールドが最小限（名前+メールのみ推奨）

### 実装パターン
```tsx
// NG: 複数のCTA
<Button>無料で始める</Button>
<Button variant="outline">デモを見る</Button>
<Button variant="link">資料請求</Button>

// OK: 1つの主要CTA + 控えめなサブ
<Button size="lg">無料で始める</Button>
<p className="text-sm text-gray-500 mt-2">
  または <a href="#demo" className="underline">デモ動画を見る</a>
</p>
```

## LIFT Model 総合スコアリング

各要素を5段階評価し、弱い項目から改善:

| 要素 | スコア (1-5) | 改善アクション |
|------|-------------|--------------|
| Value Proposition | /5 | |
| Relevance | /5 | |
| Clarity | /5 | |
| Urgency | /5 | |
| Anxiety（低いほど良い） | /5 | |
| Distraction（低いほど良い） | /5 | |
