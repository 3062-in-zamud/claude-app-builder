# A/Bテスト実験設計テンプレート

## 仮説フォーマット

```
If [変更内容]
Then [対象指標] will [改善方向（増加/減少）] by [期待変化率]
Because [根拠/ユーザー行動の仮説]
```

例:
```
If CTAボタンの色を青→オレンジに変更する
Then クリック率 will 増加 by 15%
Because オレンジは周囲のUIと対比が強く注目を集めるため
```

## サンプルサイズ計算

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| α (有意水準) | 0.05 | 偽陽性の許容率（5%） |
| β (検出力) | 0.80 | 真の差を検出する確率（80%） |
| MDE (最小検出効果) | 5-20% | 検出したい最小の改善率 |

### 簡易計算式

```
サンプルサイズ ≈ 16 × (p × (1 - p)) / MDE²

例: 現在のCVR = 5%, MDE = 20%（5% → 6%にしたい）
  = 16 × (0.05 × 0.95) / (0.01)²
  ≈ 7,600人/バリアント
```

## Guardrail Metrics

実験中に悪化させてはいけない指標:

| Guardrail | 閾値 | アクション |
|-----------|------|-----------|
| ページ読み込み時間 | +500ms以上悪化 | 実験停止 |
| エラー率 | +1%以上増加 | 実験停止 |
| バウンス率 | +5%以上増加 | 要確認 |
| 既存機能利用率 | -10%以上低下 | 要確認 |
