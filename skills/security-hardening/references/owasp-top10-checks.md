# OWASP Top 10 実装チェックリスト（Web アプリ向け）

## A01: アクセス制御の不備

- [ ] 全 API エンドポイントで認証確認
- [ ] IDOR 対策（リソースの所有者確認）
- [ ] 管理者機能へのアクセス制御
- [ ] Supabase RLS 全テーブル設定

## A02: 暗号化の失敗

- [ ] HTTPS のみ（HTTP リダイレクト）
- [ ] パスワードは bcrypt/argon2 でハッシュ化（Supabase Auth が担当）
- [ ] 機密データの暗号化（DB レベル or アプリレベル）
- [ ] JWT のアルゴリズム確認（RS256 推奨）

## A03: インジェクション

- [ ] SQL インジェクション: ORM を使用（Supabase SDK は安全）
- [ ] XSS: React/Next.js は自動エスケープ（dangerouslySetInnerHTML は禁止）
- [ ] Prompt Injection: AI 機能がある場合（システムプロンプト分離）

## A04: 安全でない設計

- [ ] 入力バリデーション: Zod でフロント+バックエンド両方
- [ ] Rate Limiting: ログイン・API エンドポイント
- [ ] エラーメッセージ: 本番環境でスタックトレース非表示

## A05: セキュリティ設定のミス

- [ ] `.env*` が `.gitignore` に含まれているか
- [ ] `NEXT_PUBLIC_*` でない変数がクライアントに露出していないか
- [ ] デフォルト認証情報が変更されているか

## A06: 脆弱で古いコンポーネント

- [ ] `npm audit --audit-level=high` で HIGH 以上なし
- [ ] Dependabot が有効
- [ ] パッケージのバージョンが最新安定版

## A07: 識別と認証の失敗

- [ ] パスワードポリシー（最低8文字以上）
- [ ] メール確認フロー
- [ ] JWT は httpOnly Cookie に格納（localStorage 禁止）
- [ ] セッションタイムアウト設定

## A08: ソフトウェアとデータの整合性の失敗

- [ ] npm パッケージの完全性確認（package-lock.json をコミット）
- [ ] CI/CD パイプラインの保護

## A09: セキュリティログとモニタリングの失敗

- [ ] Sentry でエラーをキャプチャ
- [ ] 認証失敗のログ記録
- [ ] 異常なリクエストパターンの検出

## A10: サーバーサイドリクエストフォージェリ（SSRF）

- [ ] 外部 URL へのリクエストがある場合、許可リストで制限
- [ ] ユーザー入力の URL を直接 fetch しない
