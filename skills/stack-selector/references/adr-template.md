# ADR（Architecture Decision Record）テンプレート

## 概要

ADR は技術的な意思決定を記録するための軽量ドキュメント。
「なぜこの技術を選んだか」を将来のチームメンバーが理解できるようにする。

## ファイル命名規則

```
docs/adr/NNNN-[slug].md
```

- `NNNN`: 4桁の連番（0001, 0002, ...）
- `[slug]`: ケバブケースのタイトル

## テンプレート

```markdown
# ADR-NNNN: [タイトル]
```

## ADR の運用ルール

1. **不変性**: 一度 Accepted になった ADR は変更しない。新しい決定は新しい ADR を作成し、古い ADR のステータスを `Superseded by ADR-XXXX` に変更する
2. **簡潔さ**: 1つの ADR で1つの決定のみ扱う
3. **タイミング**: 技術選定時に作成する。後付けでも可

## よくある ADR の例

| # | タイトル | カテゴリ |
|---|---------|---------|
| 0001 | Use Next.js App Router for frontend framework | フレームワーク |

| 0003 | Use Vercel for deployment | インフラ |
| 0004 | Use Tailwind CSS + shadcn/ui for styling | UI |
| 0005 | Use Vitest for unit testing | テスト |
| 0006 | Use Stripe for payment processing | 決済 |
