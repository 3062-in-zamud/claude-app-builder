# コスト予測テンプレート

## 概要

ユーザー数の増加に応じたインフラコストを予測し、
「いつ有料プランへの移行が必要か」を事前に把握する。

## ユーザー数別コスト予測テーブル

### 前提条件
```
- 1ユーザーあたり月間 PV: 50
- 1ユーザーあたり API コール: 200/月
- 1ユーザーあたり DB 容量: 5MB
- 1ユーザーあたり帯域: 50MB/月
```

### コスト予測

| ユーザー数 | Vercel | Supabase | Sentry | メール | 合計 | ユーザー単価 |
|-----------|--------|----------|--------|--------|------|-----------|
| 10 | $0* | $0 | $0 | $0 | $0 | $0 |
| 100 | $20** | $0 | $0 | $0 | $20 | $0.20 |
| 500 | $20 | $25 | $0 | $0 | $45 | $0.09 |
| 1,000 | $20 | $25 | $0 | $20 | $65 | $0.065 |
| 5,000 | $20 | $25 | $29 | $20 | $94 | $0.019 |
| 10,000 | ~$60 | ~$50 | $29 | $40 | ~$179 | $0.018 |
| 50,000 | ~$200 | ~$200 | $89 | $100 | ~$589 | $0.012 |
| 100,000 | ~$500 | ~$599 | $89 | $200 | ~$1,388 | $0.014 |

*Hobby プランは商用利用不可。テスト/開発用のみ。
**商用利用には Pro ($20/月) が必須。

### 損益分岐点

```
月額コスト ÷ ARPU = 必要ユーザー数

例: $65/月 ÷ $5/ユーザー = 13ユーザーで損益分岐
```

## コスト最適化のヒント

| 段階 | 施策 | 効果 |
|------|------|------|
| 初期 | Vercel Hobby（開発のみ） + Supabase Free | $0/月 |
| MVP | Vercel Pro + Supabase Free（注意: 7日制限） | $20/月 |
| 成長期 | ISR/SSG でサーバーレス実行を削減 | 帯域コスト削減 |
| 成長期 | 画像最適化（next/image + 外部CDN） | 帯域コスト削減 |
| スケール | Edge Functions で重い処理を分散 | レスポンス改善 |
| スケール | Supabase connection pooling | DB接続コスト削減 |
