# The Mom Test: 5原則とバッドクエスチョン例

## 概要

Rob Fitzpatrick の The Mom Test は「お母さんでも嘘をつけないような質問をする」ことで、
ユーザーインタビューの精度を劇的に向上させるフレームワーク。

## 5原則

### 原則1: 相手の人生について話す（自分のアイデアについてではない）

**Good**: 「今、請求書の管理はどうやっていますか？」
**Bad**: 「請求書管理アプリがあったら使いますか？」

ポイント: アイデアを話した瞬間、相手は「良いね」と言いたくなる。それはデータではない。

### 原則2: 具体的な過去の行動を聞く（仮定の未来ではない）

**Good**: 「最後に請求書で困ったのはいつですか？そのとき何をしましたか？」
**Bad**: 「もしこういう機能があったら使いますか？」

ポイント: 「使うと思う」は約束ではない。過去に実際に取った行動だけが信頼できる。

### 原則3: たくさん聞いて少なく話す

**Good**: （沈黙を恐れず、相手が話し終わるのを待つ）
**Bad**: 「つまりこういうことですよね？」と先回りする

ポイント: 相手が70%以上話している状態が理想。沈黙は深く考えているサイン。

### 原則4: 褒め言葉・世間話はデータにならない

**Good**: 「具体的にいくらなら払いますか？」
**Bad**: （「いいアイデアだね！」という反応を成功と見なす）

ポイント: 「すごいね」「便利そう」は社交辞令。コミットメント（時間・お金・紹介）を求める。

### 原則5: 重要な質問は怖くても聞く

**Good**: 「正直に聞きますが、この問題は本当に困っていますか？それとも我慢できるレベルですか？」
**Bad**: （聞きたくない答えが返ってきそうな質問を避ける）

ポイント: 怖い質問こそ、プロダクトの成否を分ける。早く聞くほど傷が浅い。

## バッドクエスチョン集

| バッドクエスチョン | 問題点 | 改善例 |
|------------------|--------|--------|
| 「このアプリどう思いますか？」 | アイデアを売り込んでいる | 「今どうやって解決していますか？」 |
| 「こんな機能があったら使いますか？」 | 仮定の未来を聞いている | 「最後にこの作業で困ったのはいつですか？」 |
| 「いくらなら払いますか？」（単独） | 文脈なしの価格質問 | 「今この問題にいくら使っていますか？」 |
| 「〜だと思いますよね？」 | 誘導尋問 | 「〜についてどう感じていますか？」 |
| 「みんなこう言ってるんですが」 | 社会的圧力 | （他者の意見を持ち出さない） |
| 「便利だと思いませんか？」 | Yes を誘導 | 「今の方法の何が不満ですか？」 |

## コミットメントの階段

褒め言葉ではなく、以下のコミットメントを段階的に求める:

1. **時間のコミットメント**: 「次回30分のデモを見てもらえますか？」
2. **紹介のコミットメント**: 「同じ悩みを持つ知人を紹介してもらえますか？」
3. **金銭のコミットメント**: 「事前登録で割引しますが、予約しますか？」
4. **評判のコミットメント**: 「ベータテスターとして名前を出してもいいですか？」

段階が上がるほど、相手の本気度がわかる。
