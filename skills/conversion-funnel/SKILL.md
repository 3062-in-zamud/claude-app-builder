# conversion-funnel: コンバージョンファネル戦略

## 概要

「測定されないファネルは改善されない」。AARRR Pirate Metrics フレームワークに基づき、
ユーザー獲得から収益化までの全ファネルを設計し、各段階の最適化戦略を策定します。

## ワークフロー

### Step 1: 入力ドキュメントの読み込み

以下のファイルを読み込み、ビジネスモデルとターゲットを把握:

- `docs/requirements.md` - ビジネス要件・ターゲットユーザー
- `docs/analytics-plan.md` - 計測イベント設計
- `docs/pricing-strategy.md` - 価格戦略（存在する場合）

### Step 2: AARRR Pirate Metrics ファネル定義

各段階の計測ポイントと目標を定義:

```markdown
## AARRR ファネル設計

### 1. Acquisition（獲得）
- **定義**: ユーザーがサービスを初めて認知・訪問する
- **計測イベント**: `page_viewed`, `utm_tracked`
- **チャネル**: SEO, SNS, Product Hunt, 紹介
- **目標**: 月間 [N] UU

### 2. Activation（活性化）
- **定義**: ユーザーが「Aha Moment」を体験する
- **Aha Moment**: [サービス固有の価値体験を定義]
- **計測イベント**: `sign_up_completed`, `onboarding_completed`, `first_value_achieved`
- **目標**: 訪問→登録 [N]%, 登録→活性化 [N]%

### 3. Retention（継続）
- **定義**: ユーザーが繰り返し利用する
- **計測イベント**: `return_visit`, `feature_used`
- **計測期間**: Day 1, Day 7, Day 30 リテンション
- **目標**: D7 リテンション [N]%

### 4. Referral（紹介）
- **定義**: ユーザーが他者にサービスを紹介する
- **計測イベント**: `referral_sent`, `referral_converted`
- **メカニクス**: [紹介プログラム設計]
- **目標**: バイラル係数 > 0.3

### 5. Revenue（収益）
- **定義**: ユーザーが有料プランに転換する
- **計測イベント**: `upgrade_clicked`, `payment_completed`
- **転換トリガー**: [ペイウォール設計]
- **目標**: 無料→有料転換率 [N]%
```

### Step 3: ペイウォールUI設計パターン

プロダクト特性に最適なペイウォールパターンを選定:

| パターン | 説明 | 適用場面 | 転換率目安 |
|----------|------|----------|-----------|
| Hard Paywall | 無料では使えない | 明確な価値がある専門ツール | 高（ただし獲得少） |
| Soft Paywall | 基本無料、高度な機能は有料 | SaaS、生産性ツール | 中（バランス良好） |
| Metered Paywall | 一定量まで無料、超過で有料 | API、ストレージ系 | 中〜高 |
| Feature-gated | 特定機能のみ有料 | コラボ機能、エクスポート等 | 中 |

各パターンのUI実装ガイドラインを提供:
- アップグレード促進UIの配置場所
- 制限到達時のメッセージング
- 価格表示のベストプラクティス

### Step 4: 使用量制限パターン

制限の種類と実装方針を設計:

```markdown
## 使用量制限設計

### API呼び出し数制限
- Free: [N] calls/月
- Pro: [N] calls/月
- 制限到達時: ソフトブロック + アップグレード誘導

### ストレージ制限
- Free: [N] MB
- Pro: [N] GB
- 制限到達時: 新規アップロード不可 + 既存データは維持

### 機能制限
- Free: 基本機能のみ
- Pro: [高度な機能リスト]
- 表示方法: ロック状態のUI + ツールチップで価値訴求
```

### Step 5: 紹介プログラム設計

```markdown
## 紹介プログラム設計

### インセンティブ構造
- 紹介者: [報酬（例: 1ヶ月Pro無料、クレジット付与）]
- 被紹介者: [特典（例: 初月50%OFF、延長トライアル）]

### メカニクス
- 紹介リンク生成（ユニークコード）
- 紹介追跡（Cookie + UTMパラメータ）
- 報酬付与条件（被紹介者が有料プランに転換した時点）
- 不正防止（同一IP制限、メール認証必須）

### 追跡指標
- 紹介リンク発行数
- 紹介経由登録数
- 紹介経由転換率
- バイラル係数（K-factor）
```

### Step 6: Conversion Rate 最適化チェックリスト

```markdown
## CRO チェックリスト

### ランディングページ
- [ ] ファーストビューに価値提案が明確に表示されている
- [ ] CTAボタンが目立つ配色・配置になっている
- [ ] ソーシャルプルーフ（利用者数、レビュー）がある
- [ ] ページ読み込み時間が3秒以内

### 登録フロー
- [ ] 登録ステップが3つ以内
- [ ] ソーシャルログイン（Google/GitHub）がある
- [ ] プログレスバーで進捗が見える
- [ ] 離脱時のリターゲティング設計がある

### オンボーディング
- [ ] 最初の価値体験まで5分以内
- [ ] チュートリアル/ウォークスルーがある
- [ ] テンプレート/サンプルデータで即体験可能

### アップグレード
- [ ] 制限到達時のメッセージが価値訴求している
- [ ] 年額プランの割引表示がある
- [ ] 無料トライアル期間が設定されている
```

### Step 7: ファネル漏れ分析フレームワーク

```markdown
## ファネル漏れ分析

### 分析手順
1. 各段階の転換率を計測
2. 業界ベンチマークと比較
3. 最大の漏れ（Drop-off）を特定
4. 漏れの原因仮説を3つ立てる
5. A/Bテストで検証

### ベンチマーク（SaaS業界平均）
| ファネル段階 | 業界平均 | 目標値 |
|-------------|---------|--------|
| 訪問→登録 | 2-5% | |
| 登録→活性化 | 20-40% | |
| D7 リテンション | 20-30% | |
| 無料→有料 | 2-5% | |
| 月次チャーン率 | 3-7% | |

### 優先度判定マトリクス
- **影響度**: その段階の改善が収益に与えるインパクト
- **改善容易性**: 技術的・リソース的に改善しやすいか
- **確信度**: 原因の仮説にどの程度自信があるか
```
