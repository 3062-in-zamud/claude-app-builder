# brand-foundation: ブランディング基礎の策定

## ワークフロー

### Step 1: インプット読み込み

```
docs/requirements.md を読み込む
docs/personas.md を読み込む（存在する場合のみ。存在しない場合は requirements.md のターゲットユーザー情報のみで進行する）
```

### Step 2: ブランド戦略策定（Opus で実行）

以下を決定します:

1. **プロダクト名** 3案（ユニーク・発音しやすい・ドメイン取りやすい）
2. **ブランドパーソナリティ** (例: 親しみやすい・プロフェッショナル・革新的)
   - personas.md がある場合: プライマリーペルソナの性格特性・デザイン嗜好プロファイルからブランドパーソナリティを導出する
3. **ブランドボイス** (例: カジュアル/フォーマル、ユーモア有無)
4. **カラーパレット**
   - プライマリカラー（ブランドの主張）
   - セカンダリカラー（補色）
   - ニュートラル（テキスト・背景）
   - アクセント（CTA・警告など）
5. **タイポグラフィ**
   - ヘッディング: [フォント名]
   - ボディ: [フォント名]
6. **ロゴコンセプト**
   - SVGテキストロゴを自動生成
   - 外部ツール案内: Looka.com / Canva

### Step 3: ドメイン候補提示

```bash
# 推奨ドメイン確認コマンド
whois [name1].com
whois [name1].io
whois [name1].app
```

### 出力ファイル

- `docs/brand-brief.md`

## ブランドアーキタイプ12原型

`references/brand-archetypes-12.md` に従い、プロダクトのブランドアーキタイプを選定する。

Step 2 で以下を実施:
1. requirements.md の課題・ターゲット・価値提案からアーキタイプ候補を2〜3個選出
2. personas.md の性格タイプ・UIの好みとの整合性を確認
3. 最終的に1つのプライマリアーキタイプを決定
4. アーキタイプに基づいてブランドボイス・カラー傾向を導出

## Golden Circle（Simon Sinek: Why → How → What）

ブランドの核となるメッセージを3層で定義する:

```
Why（なぜ存在するか）: [ミッション/信念]
 └─ How（どう実現するか）: [独自のアプローチ/価値提案]
     └─ What（何を提供するか）: [具体的な製品/サービス]
```

brand-brief.md に「ブランドの Golden Circle」セクションとして記載する。
Why から語ることで、機能ではなく信念で共感を得るブランドメッセージを構築する。

## StoryBrand フレーム

`references/storybrand-framework.md` に従い、ブランドストーリーを7要素で構造化する:

1. **Hero（主人公）**: ターゲットユーザー
2. **Problem（問題）**: 外的・内的・哲学的の3層で定義
3. **Guide（ガイド）**: プロダクト（共感 + 権威）
4. **Plan（計画）**: 3ステップの簡単な使い方
5. **CTA（行動喚起）**: 具体的な次のアクション
6. **Success（成功）**: 使った後の理想の状態
7. **Failure Avoidance（失敗回避）**: 使わなかった場合のリスク

LP のコピーライティングに直結するため、brand-brief.md に含める。

## カラー心理学

Step 2 のカラーパレット策定時に、色と感情の対応を考慮する:

| 色 | 感情・印象 | 適するプロダクト例 |
|----|-----------|------------------|
| 青 | 信頼・安全・プロフェッショナル | 金融、ヘルスケア、SaaS |
| 緑 | 成長・健康・自然・調和 | 環境、農業、ウェルネス |
| 赤 | 情熱・緊急・エネルギー | フード、エンタメ、セール |
| オレンジ | 友好・楽しさ・創造性 | SNS、クリエイティブツール |
| 紫 | 高級・知性・創造性 | 美容、教育、プレミアムサービス |
| 黄 | 楽観・明るさ・注意喚起 | 子供向け、レジャー |
| 黒 | 洗練・高級・モダン | ファッション、テック |
| 白 | 清潔・シンプル・余白 | ミニマルデザイン |

ペルソナの色彩の印象とアーキタイプのカラー傾向を組み合わせて最終決定する。
