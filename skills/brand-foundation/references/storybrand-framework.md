# StoryBrand フレームワーク

## 概要

Donald Miller の StoryBrand は、ブランドメッセージを「物語の構造」で整理し、
ユーザーが主人公、プロダクトがガイド（導き手）として位置づけるフレームワーク。

## 7要素の構造

```
1. Hero（主人公）── 何かを望んでいる
       │
2. Problem（問題）── 障害に直面する
       │
3. Guide（ガイド）── 導き手に出会う
       │
4. Plan（計画）── 計画を示される
       │
5. CTA（行動喚起）── 行動を促される
       │
   ┌───┴───┐
6. Success    7. Failure
   （成功）     （失敗回避）
```

## 各要素の詳細

### 1. Hero（主人公）= ターゲットユーザー

ユーザーの望みを1文で定義する。主人公はプロダクトではなくユーザー。

```
[ターゲットユーザー] は [望み/ゴール] を実現したいと思っている。
```

**NG**: プロダクトの機能を語る
**OK**: ユーザーの望みを語る

### 2. Problem（問題）= 3層で定義

| 層 | 定義 | 例（タスク管理アプリ） |
|----|------|---------------------|
| 外的問題 | 目に見える具体的な課題 | タスクが散乱して管理できない |
| 内的問題 | 課題が引き起こす感情 | 「自分はダメだ」という無力感 |
| 哲学的問題 | 「こうあるべき」という信念 | 仕事は楽しくあるべきだ |

**重要**: ユーザーが購入を決めるのは内的問題の解決のため。
外的問題だけでなく、感情レベルの課題を言語化する。

### 3. Guide（ガイド）= プロダクト

ガイドは2つの要素を持つ:

- **共感（Empathy）**: 「あなたの気持ちはわかります」
- **権威（Authority）**: 「私たちはその問題を解決できます」

```
共感: 「[内的問題] を感じている方は多いです」
権威: 「[実績/数値/専門性] で解決してきました」
```

**NG**: ガイドが主人公になる（自社の素晴らしさを語りすぎる）
**OK**: 主人公の成功を支える存在として振る舞う

### 4. Plan（計画）= 3ステップ

ユーザーの行動ハードルを下げるために、シンプルな3ステップを示す:

```
Step 1: [最初の行動]（例: 無料でアカウント作成）
Step 2: [次の行動]（例: タスクを登録する）
Step 3: [成果]（例: 仕事が整理されて安心感を得る）
```

**ポイント**: 3ステップを超えない。複雑に見えると行動しない。

### 5. CTA（行動喚起）= 2種類

| 種類 | 定義 | 例 |
|------|------|-----|
| Direct CTA | メインのコンバージョンアクション | 「無料で始める」「今すぐ登録」 |
| Transitional CTA | コミットメントが低い代替アクション | 「デモを見る」「ガイドをダウンロード」 |

**ポイント**: CTAは明確で具体的に。「詳しくはこちら」は弱い。

### 6. Success（成功）= 変容後の姿

プロダクトを使った後のユーザーの理想状態を描写する:

```
Before（使う前）:
  - [外的問題の状態]
  - [内的問題の感情]

After（使った後）:
  - [外的問題が解決した状態]
  - [内的問題が解消した感情]
  - [新しいアイデンティティ]
```

### 7. Failure Avoidance（失敗回避）= リスクの明示

行動しなかった場合のリスクを適度に示す（恐怖の煽りすぎはNG）:

```
このまま [現状の問題] を放置すると:
  - [具体的なリスク1]
  - [具体的なリスク2]
  - [感情的なリスク]
```

## brand-brief.md への記載形式

```markdown
## ブランドストーリー（StoryBrand）

### 主人公
[ターゲットユーザー] は [望み] を実現したい。

### 問題
- 外的: [具体的な課題]
- 内的: [課題が引き起こす感情]
- 哲学的: [「こうあるべき」という信念]

### ガイド
- 共感: [ユーザーの気持ちへの理解]
- 権威: [解決できる根拠]

### 計画
1. [ステップ1]
2. [ステップ2]
3. [ステップ3]

### CTA
- Direct: [メインCTA]
- Transitional: [サブCTA]

### 成功
[使った後の理想状態]

### 失敗回避
[行動しなかった場合のリスク]
```

## LP への展開

StoryBrand の7要素は、LP の構成に直接マッピングできる:

| LP セクション | StoryBrand 要素 |
|-------------|---------------|
| ヒーローセクション | Hero + Problem（外的） |
| 課題提起 | Problem（内的 + 哲学的） |
| ソリューション | Guide |
| 使い方 | Plan |
| CTA ボタン | CTA（Direct） |
| お客様の声 | Success |
| FAQ / 比較 | Failure Avoidance |
| フッターCTA | CTA（Transitional） |
