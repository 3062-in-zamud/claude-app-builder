# ポストモーテムテンプレート

## Blameless 原則

ポストモーテムは個人の責任追及ではなく、システムの改善が目的。「誰が間違えたか」ではなく「なぜシステムがそれを許したか」を問う。

---

## タイムライン

| 時刻 | イベント |
|------|---------|
| HH:MM | [検知] Sentryアラート発報 / ユーザー報告 |
| HH:MM | [確認] 問題の再現確認 |
| HH:MM | [対応開始] 原因調査開始 |
| HH:MM | [特定] 根本原因を特定 |
| HH:MM | [修正] ホットフィックス適用 / ロールバック実行 |
| HH:MM | [確認] サービス復旧確認 |
| HH:MM | [完了] 監視強化して対応終了 |

## 根本原因分析（5 Whys）

```
1. なぜサービスが停止したか？
   → DBコネクションが枯渇したため

2. なぜコネクションが枯渇したか？
   → N+1クエリが大量のコネクションを消費したため

3. なぜN+1クエリがあったか？
   → ORMのeager loadingが設定されていなかったため

4. なぜ設定されていなかったか？
   → コードレビューでパフォーマンスチェックがなかったため

5. なぜチェックがなかったか？
   → レビューチェックリストにDB最適化の項目がなかったため
```
