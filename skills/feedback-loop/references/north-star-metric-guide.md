# North Star Metric（NSM）定義・計測フレームワーク

## NSM とは

プロダクトの成長を最もよく反映する**唯一の指標**。
チーム全員が同じ方向を向くための羅針盤として機能する。

## NSM の3条件

1. **価値反映**: ユーザーが価値を感じた瞬間を計測している
2. **先行指標**: 売上やDAUより先に動く指標である
3. **行動可能**: チームのアクションで改善可能である

## プロダクトタイプ別 NSM 候補

| プロダクトタイプ | NSM 候補 | 計測例 |
|----------------|---------|--------|
| SaaS ツール | Weekly Active Users（WAU） | 週に1回以上コア機能を使ったユーザー数 |
| マーケットプレイス | 週間取引完了数 | 買い手と売り手のマッチング成立数 |
| コンテンツプラットフォーム | 週間コンテンツ消費時間 | 記事閲覧・動画視聴の合計時間 |
| コミュニティ | 週間投稿数 | コメント・投稿・リアクション数 |
| E-commerce | 週間購入完了数 | 決済完了トランザクション数 |
| 生産性ツール | 週間タスク完了数 | ユーザーがツール上で完了したタスク |
| ソーシャル | 週間アクティブな関係数 | 相互にやり取りしたユーザーペア数 |

## NSM 計測 SQL（Supabase）

```sql
-- WAU（Weekly Active Users）
SELECT COUNT(DISTINCT user_id) AS wau
FROM public.events
WHERE event_name = 'core_action'  -- コア機能の利用イベント
  AND created_at >= NOW() - INTERVAL '7 days';

-- NSM トレンド（日別）
SELECT
  DATE_TRUNC('day', d)::date AS day,
  (
    SELECT COUNT(DISTINCT user_id)
    FROM public.events
    WHERE event_name = 'core_action'
      AND created_at BETWEEN d - INTERVAL '7 days' AND d
  ) AS wau
FROM generate_series(
  NOW() - INTERVAL '30 days',
  NOW(),
  '1 day'::interval
) AS d
ORDER BY day;
```

## NSM を支えるインプット指標

NSM を分解し、改善可能なインプット指標を特定する。

```
NSM: WAU
  ├── 新規ユーザー獲得数（Acquisition）
  │     ├── ランディングページ訪問数
  │     └── サインアップ率
  ├── アクティベーション率（Activation）
  │     ├── オンボーディング完了率
  │     └── コア機能初回利用率
  └── リテンション率（Retention）
        ├── Day-1 リテンション
        └── Week-1 リテンション
```

### インプット指標の改善優先度

| 指標 | 現状 | 目標 | 改善アクション |
|------|------|------|--------------|
| サインアップ率 | [%] | [%] | CTA改善、ソーシャルログイン追加 |
| オンボーディング完了率 | [%] | [%] | ステップ削減、プログレスバー追加 |
| コア機能初回利用率 | [%] | [%] | ツールチップ、ガイドツアー |
| Week-1 リテンション | [%] | [%] | メールリマインダー、プッシュ通知 |

## アンチパターン

| NG な NSM | 理由 | 代替案 |
|----------|------|--------|
| PV（ページビュー） | 価値を反映しない虚栄指標 | コア機能の利用回数 |
| 登録ユーザー数 | 累積値は成長を反映しない | WAU |
| 売上 | 遅行指標。改善まで時間がかかる | 取引完了数 |
| セッション時間 | 長い＝良いとは限らない | タスク完了数 |
