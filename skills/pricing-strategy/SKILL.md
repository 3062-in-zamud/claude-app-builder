# pricing-strategy: 価格戦略設計

## 概要

「価格は戦略である」。プロダクトの価値に見合った価格設定を、データドリブンで設計します。
Van Westendorp 価格感度分析フレーム、Unit Economics 計算、競合価格マッピングを組み合わせ、
Free/Pro/Team の3階層プランを最適化します。

## ワークフロー

### Step 1: インプット読み込み

以下のファイルを読み込み、プロダクトの価値提案・ターゲット・競合情報を把握する:

- `docs/requirements.md` - 機能一覧・ターゲットユーザー・成功指標
- `docs/market-research.md` - 競合分析・市場規模・差別化ポイント
- `docs/personas.md`（存在する場合）- ペルソナ別の支払い意欲

### Step 2: Van Westendorp 価格感度分析

ユーザー調査用の4つの質問フレームを生成する:

```markdown
## Van Westendorp 価格感度調査テンプレート

対象: [ターゲットユーザー] に対して以下の4問を調査

1. **安すぎる価格（Too Cheap）**: いくらだと「安すぎて品質が心配」ですか？
2. **お得に感じる価格（Cheap/Good Value）**: いくらだと「お得」と感じますか？
3. **高いが許容できる価格（Expensive/High）**: いくらだと「高い」と感じますか？
4. **高すぎる価格（Too Expensive）**: いくらだと「高すぎて検討しない」ですか？

### 推奨調査方法
- Google Forms / Typeform で20〜50名に配布
- 回答をグラフ化し、Indifference Price Point（無関心価格）と Optimal Price Point（最適価格）を特定

### 仮説価格帯（調査前の推定）
- Too Cheap: $[X]/月
- Good Value: $[Y]/月
- Expensive: $[Z]/月
- Too Expensive: $[W]/月
```

### Step 3: Unit Economics 計算

```markdown
## Unit Economics モデル

### 前提条件
- 月間新規ユーザー数: [推定値]
- Free → Pro 転換率: [仮説 2〜5%]
- 月次チャーンレート: [仮説 5〜10%]

### コスト構造
| 項目 | 月額コスト | 備考 |
|------|-----------|------|
| インフラ（Vercel Pro） | $20 | |
| Supabase（Pro） | $25 | |
| ドメイン（按分） | $1 | |
| Sentry | $0〜$26 | |
| メール配信（Resend） | $0〜$20 | |
| **合計固定費** | **$[X]** | |
| ユーザーあたり変動費 | $[Y] | ストレージ・API呼び出し等 |

### 主要指標
| 指標 | 計算式 | 目標 |
|------|--------|------|
| CAC（顧客獲得コスト） | マーケティング費用 / 新規有料ユーザー数 | < $[X] |
| LTV（顧客生涯価値） | ARPU / チャーンレート | > $[Y] |
| LTV/CAC 比率 | LTV / CAC | > 3.0 |
| Payback Period | CAC / 月次ARPU | < 12ヶ月 |
| Gross Margin | (売上 - 変動費) / 売上 | > 70% |
```

### Step 4: Free/Pro/Team 3階層設計

機能分割マトリクスを作成し、各プランの価値提案を明確化する:

```markdown
## プラン設計

### 機能分割マトリクス

| 機能カテゴリ | 機能 | Free | Pro | Team |
|------------|------|------|-----|------|
| コア機能 | [基本機能1] | ✅ | ✅ | ✅ |
| コア機能 | [基本機能2] | ✅（制限付） | ✅ | ✅ |
| パワー機能 | [高度機能1] | ❌ | ✅ | ✅ |
| パワー機能 | [高度機能2] | ❌ | ✅ | ✅ |
| チーム機能 | コラボレーション | ❌ | ❌ | ✅ |
| チーム機能 | 管理者ダッシュボード | ❌ | ❌ | ✅ |
| サポート | コミュニティ | ✅ | ✅ | ✅ |
| サポート | メール | ❌ | ✅ | ✅ |
| サポート | 優先サポート | ❌ | ❌ | ✅ |

### 使用量制限

| リソース | Free | Pro | Team |
|---------|------|-----|------|
| [主要リソース1] | [制限値] | [拡張値] | 無制限 |
| [主要リソース2] | [制限値] | [拡張値] | 無制限 |
| ストレージ | [X] MB | [Y] GB | [Z] GB |
| API呼び出し | [X]/月 | [Y]/月 | 無制限 |

### プラン詳細

#### Free（$0/月）
- **目的**: プロダクトの価値を体験してもらう
- **ターゲット**: 個人ユーザー・試用者
- **制限の設計思想**: 価値は十分体験できるが、ヘビーユースには不十分

#### Pro（$[X]/月 または $[Y]/年）
- **目的**: パワーユーザーの生産性を最大化
- **ターゲット**: 個人のプロフェッショナル・フリーランサー
- **年払い割引**: 月額の[20%]OFF

#### Team（$[X]/ユーザー/月 または $[Y]/ユーザー/年）
- **目的**: チームでのコラボレーション
- **ターゲット**: 小〜中規模チーム（3〜20名）
- **最小ユーザー数**: 3名
```

### Step 5: 競合価格マッピング

WebSearch を使用して競合サービスの価格を調査:

```
検索クエリ例:
- "[競合名] pricing"
- "[競合名] plans"
- "[カテゴリ] SaaS pricing comparison {current_year}"
```

```markdown
## 競合価格マッピング

| サービス | Free | 個人プラン | チームプラン | エンタープライズ |
|---------|------|-----------|------------|----------------|
| 競合A | ✅ | $X/月 | $Y/ユーザー/月 | カスタム |
| 競合B | ❌ | $X/月 | $Y/ユーザー/月 | $Z/月 |
| 競合C | ✅（制限大） | $X/月 | $Y/ユーザー/月 | カスタム |
| **自社（提案）** | **✅** | **$X/月** | **$Y/ユーザー/月** | **-** |

### ポジショニング
- [ ] 競合より安い → 価格で勝負（コスト競争力が必要）
- [ ] 競合と同等 → 機能・体験で差別化
- [ ] 競合より高い → プレミアムポジション（ブランド力が必要）
```

### Step 6: 価格テスト計画

```markdown
## 段階的価格テスト戦略

### Phase 1: ローンチ価格（0〜3ヶ月）
- **Early Adopter 割引**: 正規価格の[30%]OFF
- **目的**: 初期ユーザーの獲得 + 支払い意欲のデータ収集
- **条件**: 「永久割引」（解約しない限り適用）

### Phase 2: 価格検証（3〜6ヶ月）
- Van Westendorp 調査を実施（Step 2 のテンプレート使用）
- A/Bテスト: 価格ページで2パターンを検証
- 指標: コンバージョン率 + ARPU の最適点を探索

### Phase 3: 価格最適化（6ヶ月〜）
- データに基づいた価格調整
- 年払いプランの導入・調整
- エンタープライズプランの検討

### 値上げ時のルール
1. 既存ユーザーは旧価格を[6ヶ月]維持（グランドファザリング）
2. 30日前にメールで通知
3. 値上げ幅は1回あたり最大20%
```

- [ ] LTV/CAC > 3 の目標が設定されているか
- [ ] 競合価格との比較が含まれているか
- [ ] Unit Economics の前提条件が明記されているか
- [ ] 段階的な価格テスト計画が含まれているか
- [ ] 既存ユーザーへの値上げ対応ルールがあるか
