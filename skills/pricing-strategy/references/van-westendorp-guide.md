# Van Westendorp 価格感度分析ガイド

## 概要

Van Westendorp Price Sensitivity Meter (PSM) は、消費者の価格許容範囲を4つの質問で特定する手法。
統計的な価格帯の「最適価格」「許容範囲」を可視化できる。

---

## 4つの質問テンプレート

### 質問フォーマット

```markdown
## 価格に関するアンケート

【製品/サービス説明】
${PRODUCT_DESCRIPTION}

---

### Q1: 高すぎて検討しない価格（Too Expensive）
「この製品が___円だとしたら、高すぎて購入を検討しないと思います」
→ 回答: ____円

### Q2: 高いが検討する価格（Expensive / Not a Bargain）
「この製品が___円だとしたら、高いとは思いますが購入を検討するかもしれません」
→ 回答: ____円

### Q3: お買い得と感じる価格（Cheap / Bargain）
「この製品が___円だとしたら、お買い得だと感じます」
→ 回答: ____円

### Q4: 安すぎて品質を疑う価格（Too Cheap）
「この製品が___円だとしたら、安すぎて品質に不安を感じます」
→ 回答: ____円
```

### 質問順序の注意点

| 推奨順序 | 質問 | 理由 |
|----------|------|------|
| 1番目 | Q3 (お買い得) | ポジティブから開始 |
| 2番目 | Q4 (安すぎる) | 下限を意識させる |
| 3番目 | Q2 (高いが検討) | 上限へ移行 |
| 4番目 | Q1 (高すぎる) | 最後に絶対上限 |

> 順序をランダム化すると回答バイアスが減少する研究もあるが、上記が最も一般的。

---

## 4つの交差点の解釈

### 累積分布グラフの構成

```
Y軸: 累積回答率 (0% - 100%)
X軸: 価格帯

プロットする4本の曲線:
- Too Cheap (累積: 低価格から高価格へ減少)
- Cheap/Bargain (累積: 低価格から高価格へ減少)
- Expensive (累積: 低価格から高価格へ増加)
- Too Expensive (累積: 低価格から高価格へ増加)
```

### 交差点の定義

| 交差点 | 略称 | 交差する曲線 | 意味 |
|--------|------|-------------|------|
| **Point of Marginal Cheapness** | PMC | Too Cheap × Expensive | これ以下だと「安すぎ」が「高い」を上回る |
| **Point of Marginal Expensiveness** | PME | Too Expensive × Cheap | これ以上だと「高すぎ」が「お買い得」を上回る |
| **Indifference Price Point** | IDP | Cheap × Expensive | 「お買い得」と「高い」が拮抗する価格 |
| **Optimal Price Point** | OPP | Too Cheap × Too Expensive | 価格抵抗が最小となる最適価格 |

### 実務での活用

```
許容価格帯 (Range of Acceptable Prices):
  PMC ─────────── OPP ─── IDP ─────────── PME
  │                │       │                │
  下限            最適    無関心            上限
  (これ以下は     価格    価格             (これ以上は
   品質不安)                               離脱増加)

推奨戦略:
- 新規参入 → OPP〜IDP の間で設定
- プレミアム → IDP〜PME の間で設定
- 浸透価格 → PMC〜OPP の間で設定
```

## サンプルサイズと実施ガイド

### 最低サンプルサイズ

| 分析タイプ | 最低サンプル | 推奨サンプル | 備考 |
|-----------|------------|------------|------|
| 単一セグメント | 100 | 200-300 | 交差点の安定に最低100必要 |
| セグメント比較 | 各100 | 各200 | セグメントあたりの数 |
| 高精度分析 | 300 | 500+ | 信頼区間を狭めたい場合 |

### 回答者の条件

```markdown
## スクリーニング基準

必須条件:
- [ ] 対象製品カテゴリの購入経験 or 購入意向がある
- [ ] 価格決定に関与する立場（B2Bの場合）
- [ ] 対象市場の地域に居住

除外条件:
- [ ] 競合企業の関係者
- [ ] 直近6ヶ月以内に同種の調査に参加
- [ ] 製品説明を理解できない（理解度チェック設問で確認）
```

## データ分析手順

### Step 1: データクリーニング

```
除外ルール:
1. Q1 < Q2 の回答（論理矛盾: 高すぎる < 高い）
2. Q4 > Q3 の回答（論理矛盾: 安すぎる > お買い得）
3. Q3 > Q2 の回答（お買い得 > 高い は矛盾）
4. 全質問同一金額の回答
5. 明らかな外れ値（市場価格の10倍以上など）

目安: クリーニング後に全体の10-15%以上除外される場合、
      質問文や製品説明の見直しが必要
```

### Step 2: 累積分布の計算

```
価格ポイントごとに累積率を計算:

Too Cheap累積(P) = (P以上と回答した数) / 有効回答数
Cheap累積(P) = (P以上と回答した数) / 有効回答数
Expensive累積(P) = (P以下と回答した数) / 有効回答数
Too Expensive累積(P) = (P以下と回答した数) / 有効回答数
```

### Step 3: 交差点の特定

```
各曲線ペアの交差を線形補間で特定:

交差点算出（隣接する2点間で符号が変わる箇所）:
  曲線A(P1) > 曲線B(P1) かつ 曲線A(P2) < 曲線B(P2) のとき
  交差価格 ≈ P1 + (P2 - P1) × (A(P1) - B(P1)) / ((A(P1) - B(P1)) + (B(P2) - A(P2)))
```

## SaaS での適用例

### 製品説明テンプレート（SaaS向け）

```markdown
【製品名】TaskFlow Pro
【カテゴリ】プロジェクト管理ツール

【含まれる機能】
- 無制限プロジェクト作成
- チームメンバー最大10名
- Slack/GitHub連携
- レポート・ダッシュボード
- 優先サポート

【課金形態】月額サブスクリプション（チーム単位）
【競合参考】Asana Business, Monday.com Pro

上記の情報を踏まえ、以下の質問にお答えください。
```

### SaaS特有の注意点

```
1. 月額 vs 年額
   → 月額で質問し、年額は別途割引率で算出
   → 年額で直接聞くと心理的アンカーが異なる

2. Per Seat vs Flat Rate
   → 1ユーザーあたりの価格で質問
   → チーム規模別にセグメント分析

3. フリーミアムの影響
   → 無料プランがある場合「安すぎる」の下限が$0に集中
   → 有料プラン限定の機能で質問するか、
     「現在無料プランをご利用の方」セグメントを分離
```

## 制限事項と補完手法

### Van Westendorp の限界

| 限界 | 対策 |
|------|------|
| 購入意向を測定しない | Gabor-Granger法と併用 |
| 競合との相対価格を考慮しない | コンジョイント分析と併用 |
| 仮想シナリオである | A/Bテストで実市場検証 |
| セグメント間の需要量が不明 | Newton-Miller-Smith拡張を適用 |

### Newton-Miller-Smith 拡張

```
各価格帯での購入確率を追加質問:

Q5: 「¥X,XXXだとしたら、購入しますか？」
  - 確実に購入する
  - おそらく購入する
  - わからない
  - おそらく購入しない
  - 確実に購入しない

→ 価格別の需要曲線を構築し、収益最大化価格を算出
```
