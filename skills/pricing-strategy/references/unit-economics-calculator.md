# Unit Economics 計算ガイド

## 概要

Unit Economics はビジネスの持続可能性を1顧客単位で評価する指標群。
SaaS/サブスクリプションビジネスでは LTV/CAC 比率が最重要KPI。

---

## 基本指標の算出式

### 1. CAC（Customer Acquisition Cost / 顧客獲得コスト）

```
CAC = 一定期間のマーケティング・営業費用合計 / 同期間の新規獲得顧客数

内訳:
  マーケティング費用:
    + 広告費（Google Ads, Meta Ads, etc.）
    + コンテンツ制作費
    + SEO/SNS運用費
    + ツール費用（HubSpot, Mailchimp, etc.）
    + マーケティング人件費（按分）

  営業費用（B2B の場合）:
    + 営業担当人件費
    + デモ/PoC費用
    + 営業ツール費用（Salesforce, etc.）
    + トラベル・イベント費用
```

#### チャネル別 CAC

```
Blended CAC = 全チャネル合計費用 / 全新規顧客数

チャネル別 CAC:
  Paid CAC   = 広告費 / 広告経由の新規顧客数
  Organic CAC = SEO・コンテンツ費用 / オーガニック経由の新規顧客数
  Referral CAC = 紹介プログラム費用 / 紹介経由の新規顧客数
```

---

### 2. LTV（Lifetime Value / 顧客生涯価値）

#### 基本算出式

```
LTV = ARPU × Gross Margin % × Average Customer Lifetime

ここで:
  ARPU (Average Revenue Per User) = 月間総収益 / アクティブ有料ユーザー数
  Average Customer Lifetime = 1 / Monthly Churn Rate
```

#### 詳細算出式（月次チャーン考慮）

```
LTV = ARPU × Gross Margin % / Monthly Churn Rate

例:
  ARPU = ¥5,000/月
  Gross Margin = 80%
  Monthly Churn = 3%

  LTV = 5,000 × 0.80 / 0.03 = ¥133,333
```

#### コホート別 LTV（より正確）

```
コホート LTV(n月) = Σ(月t=0→n) [ARPU(t) × Retention(t) × Gross Margin]

ここで:
  ARPU(t) = 月tでの平均単価（アップセルで増加しうる）
  Retention(t) = 月tでの残存率

12ヶ月 LTV = Σ(t=0→11) [ARPU(t) × Retention(t) × GM]
```

---

### 3. LTV/CAC 比率

```
LTV/CAC Ratio = LTV / CAC

判定基準:
  < 1.0  → 赤字。ビジネスモデルの見直しが急務
  1.0-3.0 → 投資回収は可能だが効率が悪い
  3.0-5.0 → 健全。SaaSの目標ライン ★
  > 5.0  → 良好だが、成長投資が不足している可能性
```

#### 業界ベンチマーク

| ステージ | LTV/CAC 目安 | 備考 |
|---------|------------|------|
| シード期 | 1.5-3.0 | PMF探索中は低くて可 |
| シリーズA | 3.0+ | 投資家の最低ライン |
| グロース期 | 3.0-5.0 | 効率と成長のバランス |
| 成熟期 | 5.0+ | マーケティング投資余地あり |

---

### 4. Payback Period（投資回収期間）

```
Payback Period (月) = CAC / (ARPU × Gross Margin %)

例:
  CAC = ¥30,000
  ARPU = ¥5,000/月
  Gross Margin = 80%

  Payback = 30,000 / (5,000 × 0.80) = 7.5ヶ月
```

#### 判定基準

| Payback Period | 評価 |
|---------------|------|
| < 6ヶ月 | 優秀 |
| 6-12ヶ月 | 良好（SaaS標準） |
| 12-18ヶ月 | 要改善 |
| > 18ヶ月 | 危険。キャッシュフロー圧迫 |

---

### 5. Gross Margin（売上総利益率）

```
Gross Margin % = (Revenue - COGS) / Revenue × 100

SaaS の COGS に含めるもの:
  + ホスティング費用（AWS/GCP/Vercel）
  + サードパーティAPI費用（OpenAI, Stripe手数料, etc.）
  + カスタマーサポート人件費（直接対応分）
  + オンボーディング費用
  + データベース費用

SaaS の COGS に含めないもの:
  - 開発人件費（R&D → 営業費用）
  - マーケティング費用
  - 管理部門費用
```

#### SaaS Gross Margin ベンチマーク

| Gross Margin | 評価 |
|-------------|------|
| > 80% | 優秀（Pure SaaS） |
| 70-80% | 良好 |
| 60-70% | 普通（サービス要素あり） |
| < 60% | SaaS としては低い |

## 改善レバーの優先順位

### LTV/CAC < 3.0 の場合

```
改善インパクト順:

1. チャーン削減 (LTV向上)
   - 最も効果が大きい
   - Churn 5% → 3% で LTV が 67% 向上
   → オンボーディング改善、機能充実、CS強化

2. ARPU 向上 (LTV向上)
   - アップセル / クロスセル
   - 価格改定（値上げ）
   - 上位プランの設計
   → 利用量ベース課金、機能制限の見直し

3. CAC 削減
   - チャネルミックス最適化
   - コンバージョン率改善
   - オーガニック比率向上
   → LP改善、リファラル強化、コンテンツSEO

4. Gross Margin 改善
   - インフラコスト最適化
   - サポート自動化
   - API費用の交渉/代替
```
