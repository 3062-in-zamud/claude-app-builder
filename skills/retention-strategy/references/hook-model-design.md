# Hook Model 設計ガイド（Nir Eyal）

## 4フェーズ

```
Trigger → Action → Variable Reward → Investment
   ↑                                      │
   └──────────────────────────────────────┘
              (ループ)
```

## 1. Trigger（きっかけ）

### 外部トリガー
- メール通知（Day 7: 「新しいコメントが届いています」）
- プッシュ通知（「{ユーザー名}があなたの投稿にリアクションしました」）
- SNS共有（「{ユーザー名}が{成果}を達成しました」）

### 内部トリガー（目標）
- 退屈 → 「何か面白いものないかな」→ アプリを開く
- 不安 → 「データを確認しなきゃ」→ ダッシュボードを見る
- 達成欲 → 「今日のタスクを完了したい」→ ログイン

## 2. Action（最小行動）

**Fogg Behavior Model**: B = M × A × T

| 要素 | 意味 | 最適化方法 |
|------|------|-----------|
| Motivation (M) | やる気 | 価値を明示、Social Proof |
| Ability (A) | 能力（簡単さ） | ステップ数削減、デフォルト設定 |
| Trigger (T) | きっかけ | 適切なタイミングの通知 |

最小行動の例:
- 1クリックでいいね
- スワイプで完了
- ワンタップでログ記録

## 3. Variable Reward（可変報酬）

| 種類 | 説明 | 例 |
|------|------|-----|
| Tribe（社会的報酬） | 他者からの承認 | いいね、コメント、フォロワー増 |
| Hunt（情報報酬） | 新しい発見 | フィード更新、おすすめ |
| Self（自己達成報酬） | 達成感 | レベルアップ、ストリーク、バッジ |

**可変**であることが重要（毎回同じだと飽きる）。

## 4. Investment（投資）

ユーザーがプロダクトに蓄積する価値:
- データ: 履歴、設定、カスタマイズ
- コンテンツ: 投稿、ファイル、プロジェクト
- 社会的資本: フォロワー、コネクション
- スキル: 使い方の習熟

投資が増えるほど離脱コストが上がる（= リテンション向上）。

## プロダクトタイプ別設計例

### タスク管理アプリ
- Trigger: 朝の通知「今日のタスク3件」
- Action: タスクをタップして完了
- Reward: 完了アニメーション + 達成率表示（Self）
- Investment: タスク履歴、プロジェクト構造

### SNSアプリ
- Trigger: 「{名前}があなたの投稿にコメントしました」
- Action: 通知タップ → コメント確認
- Reward: コメント内容（Tribe）+ 新着フィード（Hunt）
- Investment: 投稿、フォロワー、プロフィール
